## 1. List Resources

```
k get all            	            # List all major resources in the current namespace
k get all -A                      # List all resources across ALL namespaces
k get <resource>                  # List a specific resource type (pods, svc, deploy, rs, etc.)
k get <resource> -o wide          # List resources with more details (IP, Node, etc.)
```


## 2. Inspect Resource Details

```
k describe <resource> <name>      # Show detailed information about a specific resource
```

## 3. Logs (Pod Logs)

```
k logs <pod-name>                 # View logs of the Pod
k logs -f <pod-name>              # Follow (live stream) Pod logs in real-time
```

## 4. Execute Inside Pod

```
k exec -it <pod-name> -- sh       # Open an interactive shell inside a Pod's container
```

## 5. Create / Apply / Edit Resources

```
k apply -f <file.yaml>            # Apply configuration (create or update) from YAML
k create -f <file.yaml>           # Create a resource from a YAML file
k edit <resource> <name>          # Edit the live configuration of a running resource
```

## 6. Delete Resources

```
k delete <resource> <name>        # Delete resource by type and name
```

## 7. Resource Usage (Metrics)

```
k top pod                         # Show CPU/Memory usage of Pods
k top node                        # Show CPU/Memory usage of Nodes
```

## List Deployments

```
k get deploy                                        # List all Deployments
```

## Rollout (Deployment lifecycle)

```
k rollout status deploy/<name>                      # Check rollout status (progress of Deployment update)
k rollout undo deploy/<name>                        # Rollback Deployment to previous revision
```

## Update Deployment Image

```
k set image deploy/<name> <container>=<new-image>   # Update container image in a Deployment
```

## Scale Deployment

```
k scale deploy/<name> --replicas=N                  # Scale the Deployment to N replicas
```

## Autoscale Deployment

```
k autoscale deploy <name> --max=10                  # Create an HPA (autoscaler) with max 10 pods
```

## Inspect Node Scheduling State

```
k describe node <node-name>                         # Check existing taints, conditions, resources
```

## Taints (Control what pods can run on the node)

```
k taint node <node-name> key=value:effect           # Apply a taint to block or control pod scheduling
                                                    # Effects: NoSchedule | PreferNoSchedule | NoExecute
k taint node <node-name> key:effect-                # Remove a specific taint (the trailing '-' is required)
```

## Check Tolerations on Pods

```
k get pods -o yaml | grep tolerations -A 5          # View tolerations applied to pods
```

## Node Scheduling Control (Cordon / Uncordon / Drain)

```
k cordon <node-name>                                # Mark a node as unschedulable (no new Pods will be scheduled)
k uncordon <node-name>                              # Mark node as schedulable again
k drain <node-name>                                 # Safely evict all pods for maintenance (daemonsets unaffected)
```

## Node Scheduling Controls

```
k cordon <node-name>                                        # Mark node unschedulable (no new pods)
k drain <node-name>                                         # Evict pods, prepare for maintenance
k drain <node-name> --ignore-daemonsets                     # Drain but skip DS pods (recommended)

k uncordon <node-name>                                      # Make node schedulable again
k delete node <node-name>                                   # Remove node object (used after shutdown)
```
